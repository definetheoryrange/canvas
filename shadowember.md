# Bifrost Navigator

Bifrost Navigator 是一个面向技术决策者与架构师的高质量外链资源导航与状态监控项目。本项目不直接托管文档或镜像，而是通过人工筛选与自动化巡检相结合的方式，维护一份面向企业级基础设施、云原生生态、安全合规及开源治理领域的权威外部资源清单。其核心价值在于帮助团队降低信息筛选成本，规避因第三方域名变更、证书过期或内容策略调整导致的构建与合规风险，同时为内部技术选型提供可追溯的参考基线。

本项目定位为技术基础设施的“前端哨站”，通过对特定域名集合进行可用性、协议合规性及内容变更感知的周期性检测，为运维、安全及开发团队提供结构化的外链状态视图。Bifrost Navigator 本身不提供代理或镜像服务，所有链接均指向原始第三方站点，用户需自行遵守各站点的使用条款。

## 功能概览

- **外链资源清单版本化管理** 提供按批次、按主题组织的结构化外链列表，每条记录包含原始域名、协议类型及首次收录日期，支持与内部 CMDB 或监控系统对接。

- **HTTPS 证书有效期轮询** 对收录的域名执行每日定时检测，自动计算证书剩余天数，并在到期前 15 天通过 Webhook 输出告警事件，便于团队提前更新信任链。

- **DNS 解析与运营商可达性探测** 从多个地理位置分布式节点发起 A/AAAA 记录查询及 TCP 握手检测，识别域名劫持、解析超时或区域性阻断现象，结果以结构化 JSON 形式输出。

- **页面内容哈希指纹比对** 对每个目标 URL 的根页面进行定期请求，计算响应体 SHA-256 哈希值，当指纹发生变化时触发差异报告，用于感知页面改版或内容策略调整。

- **自定义标签与分类过滤** 允许用户为每个链接附加标签（如 `infra`, `security`, `cn`, `official`），支持按标签组合筛选生成不同用途的视图，例如仅输出国内可直连的官方域名列表。

- **历史变更审计日志** 所有检测事件、状态变更及标签修改操作均写入不可变审计日志，满足 ISO 27001 及 SOC2 合规审计对第三方依赖变更追溯的要求。

- **RESTful API 与 Prometheus Exporter** 提供标准 HTTP API 供 CI/CD 流水线调用获取最新状态，同时暴露 Prometheus 指标接口，便于集成至现有监控面板。

## 应用场景

- **企业私有仓库镜像配置验证** 当团队维护内部 PyPI、NPM 或 Docker Registry 镜像加速器时，需要确保上游源站域名的可用性与证书有效性。Bifrost Navigator 可定期校验相关域名，并在配置变更前提供预检查报告。

- **安全合规策略基线审计** 安全团队需要定期审查业务系统依赖的所有外部域名是否遵循公司出站策略。本项目提供的标签化清单与可达性数据可作为策略违规检测的输入源，减少人工整理耗时。

- **开源组件供应链风险评估** 架构师在引入开源组件时需评估其官方网站、文档站及社区论坛的活跃度与可访问性。项目维护的结构化列表可作为选型报告中的“外部依赖健康度”附件，提升决策透明度。

- **多云环境下的统一出站白名单管理** 针对部署在多个公有云或混合云环境的应用，网络团队需维护统一的出站防火墙白名单。本项目的周期性探测结果可自动同步至 IP 地址管理平台，辅助识别已失效或已变更的域名所对应的 IP 段。

## 快速开始

以下步骤适用于 Linux/macOS 环境，要求已安装 Git、Go 1.21+ 及 Docker Compose 可选。

```bash
# 1. 克隆代码仓库
git clone https://github.com/bifrost-navigator/navigator.git
cd navigator

# 2. 安装依赖并构建二进制文件
make deps
make build

# 3. 运行基础探测任务（默认使用内置批次 7/8 资源列表）
./bin/navigator probe --batch 7 --output json > report_batch7.json

# 4. （可选）启动 API 服务与定时调度器
./bin/navigator serve --config configs/production.yaml
```

## 常见问题

**Q：探测任务是否会对我所访问的第三方网站造成压力？**

A：每个目标域名默认每 24 小时仅发起一次 GET 请求（仅请求根路径 `/`，且携带 `User-Agent: Bifrost-Navigator/1.0`），请求超时设置为 10 秒，不执行 JavaScript 或下载子资源。该流量级别相当于普通用户的单次页面访问，对绝大多数站点的负载影响可忽略不计。如果您是某个收录站点的管理员并希望调整探测频率或彻底移除，请通过 GitHub Issues 联系我们并提供验证身份的公开信息。

**Q：项目如何处理某些域名在中国大陆地区无法访问的情况？**

A：本项目仅如实报告从特定探测节点发起的连接结果，并不提供任何代理、加速或镜像服务。对于因网络策略导致的超时或重置，系统会在输出报告的 `reachability` 字段中标记为 `blocked` 或 `timeout`，并将状态变更写入审计日志。用户应根据自身网络环境判断是否将该域名纳入内部白名单或寻求合规的替代访问方式。

**Q：如何将新的外部资源链接加入监控清单？**

A：您可以通过两种方式扩展监控范围。其一，在运行时通过 API `/api/v1/batches` 提交自定义批次定义（JSON 格式），该批次会与内置清单合并；其二，在项目根目录下的 `internal/batch/custom.yaml` 文件中添加条目，然后重新构建并启动服务。请注意，自定义批次仅存储在本地存储中，不会自动合并至官方资源列表，除非您提交 Pull Request 并经过审核。
