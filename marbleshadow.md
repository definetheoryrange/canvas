# Bifrost Gateway Index

Bifrost Gateway Index 是一个面向企业级网络架构师、系统运维工程师及安全审计人员的技术资源导航工具。该项目不直接提供代理或隧道功能，而是通过结构化整理与实时可用性监测，为用户提供高质量的边缘网络网关、官方镜像站与软件分发渠道的元数据索引。其核心价值在于帮助技术团队在复杂的网络环境中快速定位可信的软件源与官方入口，降低因域名变更或访问策略调整导致的运维中断风险。

本项目定位于技术资源的外链汇总层，不对任何外部站点的内容进行修改或代理。所有收录的链接均经过基础可用性校验，并辅以定期的访问延迟与证书有效性检测，确保索引数据的实时性与可靠性。Bifrost Gateway Index 适用于企业内部文档站、运维知识库以及开发环境初始化脚本中的源地址管理场景。

## 功能概览

- **结构化资源目录**：按机构类型与域名后缀对收录的网关地址进行逻辑分组，便于用户根据网络策略选择最优入口。
- **可用性健康检查**：内置轻量级 HTTP/HTTPS 探活机制，周期性验证各索引地址的响应状态码与 TLS 证书有效期，异常时输出告警日志。
- **Markdown 原生集成**：所有资源链接以代码块形式呈现，可直接嵌入技术文档、运维手册或自动化部署脚本的注释中。
- **批量导入与导出**：支持通过 YAML 配置文件批量添加或移除索引条目，适配动态变化的网络环境。
- **访问统计摘要**：记录各链接的最近七日访问成功率与平均响应时间，辅助运维决策。
- **离线缓存模式**：在无法直连外部网络的环境中，支持加载本地缓存的索引快照，确保文档导航功能不中断。
- **策略规则标签**：允许用户为每个链接自定义标签（如 `official`、`mirror`、`backup`），便于按策略过滤输出结果。

## 应用场景

- **企业内部运维文档站**：将 Bifrost Gateway Index 集成至 Confluence 或 MkDocs 文档系统中，为开发团队提供统一且经过验证的官方网关入口列表，避免因错误地址导致部署失败。
- **自动化部署前置检查**：在 CI/CD 流水线中加入索引健康检查步骤，若检测到关键网关地址不可达，则自动触发告警并暂停发布流程，防止运行时环境异常。
- **网络隔离区资源映射**：在需要严格管控出口流量的安全网络中，使用本索引作为白名单地址的初始参考基线，结合防火墙策略进行精细化放行配置。
- **技术培训与新人指导**：作为网络基础架构培训材料的一部分，帮助新入职工程师快速理解企业所使用的边缘网关体系及备选入口机制。

## 快速开始

以下操作步骤适用于 Linux 与 macOS 环境，确保系统已安装 Git 及 Node.js 18.0 以上版本。

```bash
# 1. 克隆项目仓库
git clone https://github.com/bifrost-index/gateway-index.git
cd gateway-index

# 2. 安装项目依赖
npm install

# 3. 启动本地索引服务（开发模式）
npm run dev

# 4. 执行完整可用性检查（可选）
npm run check:all

# 5. 构建生产环境静态站点
npm run build
```

访问启动后输出的本地地址（通常为 `http://localhost:3000`）即可查看当前索引资源列表与健康状态面板。

## 常见问题

**Q1：部分索引地址返回超时或连接拒绝，该如何处理？**

A1：首先确认本地网络环境是否能够直连目标域名，尝试使用 `curl -v` 手动测试。若确认为目标服务临时不可用，索引面板将在下一次探测周期（默认 5 分钟）后自动更新状态。若持续不可用超过 24 小时，请通过 Issue 反馈，维护者会核实并从索引中标记或移除该条目。

**Q2：如何在离线内网环境中使用本索引？**

A2：在具备网络访问的环境中预先执行 `npm run build` 构建静态站点，并将生成的 `dist` 目录整体打包传输至内网服务器。随后可通过 `npm run serve:offline` 启动离线模式，该模式将使用构建时缓存的健康快照，不再执行外部探测。

**Q3：是否支持通过代理服务器执行可用性检查？**

A3：支持。在 `config/policy.yaml` 中设置 `proxy` 字段，填写代理地址（如 `http://proxy.example.com:8080`），所有探测请求将自动经由该代理发出。同时支持 `no_proxy` 列表，用于绕过内网地址。
